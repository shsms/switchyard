    Switchyard reference
    
        
      
    
      
    
        
    Modes
    
        
      
          
    • The chrome's Dashboard / Topology buttons swap the main pane. The REPL drawer + side panel stay visible in both. Press 1 for Dashboard, 2 for Topology.
      • 
          
    • Mode preference persists in localStorage; refresh restores the last-picked mode.
      • 
        
    
        
    Dashboard
    
        
      
          
    • Pool tiles (top row) — grid, battery pool, PV, consumer, producer. Each tile renders a 60-second sparkline + the formula tree on click.
      • 
          
    • Battery pool envelope — lower → upper bounds line under the pool reading, derived from BatteryPool::power_bounds.
      • 
          
    • Tier-2 rows (Batteries) — name + health pill + SoC bar + DC W. Rows sort SoC ascending. Click a row to flip to Topology with the node selected.
      • 
          
    • Tier-3 rows (Inverters) — name + subtype + health + envelope visual (lower / measured / upper). Rows pinned at either bound float to the top.
      • 
          
    • Tier-5 rows (EV / CHP) — name + health + SoC (EV only) + power readout.
      • 
        
    
        
    Aggregations
    
        
      
          
    • Pool tiles read from the same frequenz-microgrid client a production control app would use. Click any tile to open the formula tree in the side panel — each #N is a link back to the topology canvas.
      • 
          
    • Pulse-bar pills — graph (✓ / ⚠) summarises validation warnings from frequenz-microgrid-component-graph; loopback is the Microgrid client status; density + TZ toggle visual options.
      • 
        
    
        
    Snapshots
    
        
      
          
    • Snapshots button (top right) — saves the current overrides file under snapshots/<name>.lisp; load to replay that state. Headless: swctl snapshot save / load / list.
      • 
        
    
        
    Topology canvas
    
        
      
          
    • Click a node — inspect it (charts + overrides + knobs in the side panel).
      • 
          
    • Shift+Drag on empty canvas — rubber-band a selection rectangle; nodes inside the box are selected.
      • 
          
    • Shift+Click a node — toggle that node into / out of the current selection (after a rubber-band, or to build one a node at a time).
      • 
          
    • Ctrl/Cmd+hold + drag from one node to another — create an edge between them.
      • 
          
    • Right-click — context menu. With a selection: Copy, Cut, Delete. With nothing selected and a non-empty clipboard: Paste. Right-clicking a node not in the current selection resets the selection to that node first.
      • 
          
    • Ctrl/Cmd+C — copy selection. Ctrl/Cmd+V — paste. Ctrl/Cmd+X — cut. Each paste re-uses the last copy; clipboard captures structure only (categories + internal edges), so clones use category defaults.
      • 
          
    • Del / Backspace — delete selection. Ctrl/Cmd+A — select all visible. Esc — deselect.
      • 
        
    
        
    Side panel + chrome
    
        
      
          
    • Add component (lower side panel) — pick a category, click Create; an auto-id'd component appears. Wire it via Ctrl/Cmd+drag.
      • 
          
    • Defaults — per-category defaults (Lisp values); edits write to the override file.
      • 
          
    • Overrides pill — count of forms in config.ui-overrides.<microgrid-id>.lisp. Every successful canvas / REPL edit appends to that file immediately. Click the pill to open the dialog: tick the rows you want gone, hit Delete selected; the file rewrites and the world reloads in one round trip.
      • 
        
    
        
    REPL
    
        
      
          
    • Ctrl/Cmd+Enter — eval. Tab — complete.
      • 
          
    • Drag the splitter between the topology row and the drawer to resize.
